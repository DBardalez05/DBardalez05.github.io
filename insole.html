Smart Insole Performance Feedback System

Built a wearable smart insole prototype that used force-sensitive resistors and an ESP32 to classify heel strike, neutral stance, and forefoot loading in real time for athletic form feedback.

Recognition: Babson Buildathon - 1st Place, AI-Enhanced Athletic Performance Track
Timeline: Built and demonstrated within a 6-hour buildathon sprint

Overview

This project explored how embedded sensing could be used to give athletes fast, understandable feedback on foot-loading patterns during movement. Our team built a smart insole prototype that measured pressure at key points of the foot and streamed the data wirelessly for downstream visualization and coaching.

The goal was not to build a medical-grade biomechanics platform, but to create a working wearable demo that could distinguish simple states such as heel-dominant loading, neutral loading, and forefoot loading. That made it a strong fit for real-time athletic feedback and rapid prototyping under tight time constraints.

What I Built

  • Validated individual FSR sensors and tuned the analog readout setup to get usable signals.
  • Integrated the sensing hardware with an ESP32 for embedded real-time data collection.
  • Implemented threshold-based logic to classify heel strike, neutral stance, and forefoot loading.
  • Brought up wireless data serving from the ESP32 so the frontend could read live data.
  • Transitioned the circuit from breadboard to a battery-powered protoboard for untethered use.
  • Adapted the sensing layout during prototyping after sensor loss and limited resistor availability.

System Architecture

Sensing

Four force-sensitive resistors were placed under key foot regions to capture simple plantar loading patterns.

Embedded Processing

An ESP32 handled analog sensor acquisition, basic signal conditioning, and on-device state classification.

Wireless Output

The board served live data wirelessly so an external interface could interpret the state and visualize results.

Wearable Prototype

The final prototype ran from portable power and was mounted on protoboard to support a fully untethered demo.
